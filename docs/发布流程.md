# 发布流程

## 版本号

当前版本号在以下位置维护：

- `src/*/Properties/AssemblyInfo.cs`
- `installer/AiBatchRenamer.iss`

发布前应统一更新 `AssemblyVersion`、`AssemblyFileVersion` 和 Inno Setup 中的 `MyAppVersion`。

## Windows 构建

在 Windows 开发机执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build-windows.ps1 -RunTests -BuildInstaller
```

期望输出：

- `src\AiBatchRenamer.App\bin\Release\AiBatchRenamer.exe`
- `installer\output\AiBatchRenamer-Setup-<version>.exe`

安装包当前使用 Inno Setup 默认语言，以避免 CI 环境缺失额外语言文件导致构建失败。

## GitHub Actions

推送到 GitHub 后，`Windows Build` 工作流会：

1. 构建解决方案
2. 运行 `AiBatchRenamer.Tests.exe`
3. 安装 Inno Setup
4. 生成安装包
5. 上传应用目录和安装包 artifacts

## GitHub Releases 下载页

右侧 `Releases` 区域显示的是 GitHub Release，不是普通 CI artifact。

发布新版本时，推送 `v*` 标签即可触发 `Publish Release` 工作流：

```powershell
git tag v0.1.0
git push origin v0.1.0
```

工作流会自动构建、测试、生成安装包，并上传到 Releases：

- `AiBatchRenamer-Setup-<version>.exe`
- `AiBatchRenamer-Portable-<version>.exe`
- `SHA256SUMS.txt`

也可以在 GitHub 的 `Actions` 页面手动运行 `Publish Release`，输入版本标签，例如 `v0.1.0`。

## 发布前检查

- 确认 DeepSeek API Key 没有进入仓库
- 确认 `PRIVACY.md` 与实际发送数据一致
- 在 Windows 7 SP1 + .NET Framework 4.8、Windows 10、Windows 11 上手动验证
- 测试中文路径、中文文件名、撤销、递归添加文件夹、模板命名、CSV 导出
