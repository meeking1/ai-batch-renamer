# 验证清单

## 构建验证

在 Windows 开发机执行：

```powershell
msbuild AiBatchRenamer.sln /p:Configuration=Release
```

或：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build-windows.ps1
```

期望：

- 解决方案构建成功
- 生成 `src\AiBatchRenamer.App\bin\Release\AiBatchRenamer.exe`

## 功能验证

### 添加文件

- 点击“添加文件”选择多个文件
- 拖拽多个文件到窗口
- 拖拽文件夹到窗口

期望：

- 文件出现在表格中
- 重复文件不会重复添加

### 多名称替换

1. 选择“多名称逐行替换”
2. 输入与文件数量相同的多行名称
3. 点击“生成预览”

期望：

- 每个文件对应一行名称
- 保留原扩展名
- 状态为“可执行”

### 数量不足

1. 文件数量多于名称行数
2. 点击“生成预览”

期望：

- 缺少名称的行状态为“无效”
- 不能执行重命名

### 查找替换

1. 选择“查找替换”
2. 输入查找内容和替换内容
3. 点击“生成预览”

期望：

- 原文件名中的匹配文本被替换
- 无冲突时可执行

### DeepSeek 自然语言命名

1. 在右侧输入 DeepSeek API Key
2. 模型保持 `deepseek-v4-flash`
3. 点击“保存 AI 设置”
4. 选择“自然语言规则”
5. 输入自然语言命名要求
6. 点击“生成预览”

期望：

- 状态栏显示正在调用 DeepSeek
- 表格生成新文件名预览
- API Key 不会显示在界面或明文配置中

也可以设置环境变量：

```powershell
setx DEEPSEEK_API_KEY "你的 DeepSeek API Key"
```

重新打开程序后，界面应提示已从环境变量读取 API Key。

### 冲突检测

输入重复的新名称。

期望：

- 重复项状态为“冲突”
- 不能执行重命名

### 撤销

1. 完成一次成功重命名
2. 点击“撤销上次操作”

期望：

- 文件恢复到原名称
- 再次撤销时提示没有可撤销操作

## 兼容验证

必须人工验证：

- Windows 7 SP1 + .NET Framework 4.8
- Windows 10 22H2
- Windows 11 当前稳定版

重点检查：

- 程序启动
- 中文路径
- 中文文件名
- 非管理员权限
- 文件被占用时的失败提示
- 长路径提示
