# 验证清单

## 构建验证

在 Windows 开发机执行：

```powershell
msbuild AiBatchRenamer.sln /p:Configuration=Release
```

或：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build-windows.ps1
```

带测试：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build-windows.ps1 -RunTests
```

带安装包：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build-windows.ps1 -RunTests -BuildInstaller
```

期望：

- 解决方案构建成功
- 生成 `src\AiBatchRenamer.App\bin\Release\AiBatchRenamer.exe`
- `AiBatchRenamer.Tests.exe` 输出全部测试通过
- GitHub Actions 中的 `Windows Build` 工作流可在 Windows runner 上编译并上传 Release 产物
- GitHub Actions 上传 `AiBatchRenamer-Setup` 安装包产物

## 功能验证

### 添加文件

- 点击“添加文件”选择多个文件
- 拖拽多个文件到窗口
- 拖拽文件夹到窗口
- 勾选“添加文件夹时包含子文件夹”后添加包含子目录的文件夹

期望：

- 文件出现在表格中
- 重复文件不会重复添加
- 递归模式会包含子文件夹中的文件

### 文件列表排序

- 选中一个或多个文件
- 点击“上移”或“下移”
- 选择排序方式并点击“应用排序”
- 点击“移除选中”

期望：

- 文件顺序按操作变化
- 序号重新计算
- 多名称替换按新顺序匹配
- 自动排序可按文件名、修改时间、大小改变顺序
- 移除选中文件后列表数量减少

### 多名称替换

1. 选择“多名称逐行替换”
2. 输入与文件数量相同的多行名称
3. 点击“生成预览”

期望：

- 每个文件对应一行名称
- 保留原扩展名
- 状态为“可执行”

### 数量不足

1. 文件数量多于名称行数
2. 点击“生成预览”

期望：

- 缺少名称的行状态为“无效”
- 不能执行重命名

### 查找替换

1. 选择“查找替换”
2. 输入查找内容和替换内容
3. 点击“生成预览”

期望：

- 原文件名中的匹配文本被替换
- 无冲突时可执行

### 模板命名

1. 选择“模板命名”
2. 输入 `{folder}-{name}-{index:000}`
3. 点击“生成预览”

期望：

- 新文件名按当前顺序生成编号
- 原文件名主体和所在文件夹名可正确替换
- 默认保留原扩展名

### 导出预览和日志

1. 生成任意预览
2. 点击“导出预览”
3. 保存 CSV
4. 点击“打开日志”

期望：

- CSV 包含原文件名、新文件名预览、状态、路径
- 日志目录可打开

### DeepSeek 自然语言命名

1. 在右侧输入 DeepSeek API Key
2. 模型保持 `deepseek-v4-flash`
3. 点击“保存 AI 设置”
4. 点击“测试 DeepSeek”
5. 选择“自然语言规则”
6. 输入自然语言命名要求
7. 点击“生成预览”

期望：

- 测试连接成功时状态栏显示示例名称
- 状态栏显示正在调用 DeepSeek
- 表格生成新文件名预览
- API Key 不会显示在界面或明文配置中

也可以设置环境变量：

```powershell
setx DEEPSEEK_API_KEY "你的 DeepSeek API Key"
```

重新打开程序后，界面应提示已从环境变量读取 API Key。

### 冲突检测

输入重复的新名称。

期望：

- 重复项状态为“冲突”
- 不能执行重命名

### Windows 保留名检测

输入 `CON`、`PRN`、`AUX`、`NUL`、`COM1` 或 `LPT1` 作为新名称。

期望：

- 对应行状态为“无效”
- 不能执行重命名

### 撤销

1. 完成一次成功重命名
2. 点击“撤销上次操作”
3. 完成多次重命名后，点击“刷新历史”
4. 选择一条历史并点击“撤销选中历史”

期望：

- 文件恢复到原名称
- 再次撤销时提示没有可撤销操作
- 最近历史可在下拉框中选择
- 撤销选中历史会尝试恢复该历史中的成功项

## 兼容验证

必须人工验证：

- Windows 7 SP1 + .NET Framework 4.8
- Windows 10 22H2
- Windows 11 当前稳定版

重点检查：

- 安装和卸载
- 未安装 .NET Framework 4.8 时，安装包提示缺少运行时
- 程序启动
- 中文路径
- 中文文件名
- 非管理员权限
- 文件被占用时的失败提示
- 长路径提示
